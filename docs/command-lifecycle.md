# Жизненный цикл команд

Android, API и Web UI создают только команды из allowlist. Новая команда имеет статус `queued`, источник (`api` или `web`) и срок жизни пять минут.

При следующем опросе agent получает команды и сервер переводит их в `sent`, заполняя `picked_at` и `retry_count`. Результат agent переводит команду в `success` или `failed`, записывает `completed_at`, `result` и `last_error`.

Перед опросом agent сервер помечает просроченные `queued`, `sent` и `running` команды как `expired`. Опасные действия, включая reboot и изменение SSID, требуют подтверждения в Android; в Web UI команда отображается в истории и выполняется при следующем опросе агента.
